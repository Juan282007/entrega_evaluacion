# ADR-005: Usar Docker Compose para levantar el entorno de desarrollo

## Fecha
2026-04-14

## Estado
Aceptado

## Contexto
El proyecto necesita que cualquier persona del equipo pueda levantar la base de 
datos y ejecutar las migraciones de Liquibase en su máquina local sin configuraciones 
manuales complejas ni diferencias entre entornos.

## Problema
Sin un entorno estandarizado, cada desarrollador configura su base de datos de 
forma diferente, lo que genera inconsistencias entre máquinas y dificulta reproducir 
errores o validar cambios de manera confiable.

## Decisión
Se usará Docker Compose para orquestar los servicios esenciales del proyecto:

- Servicio `postgres` — levanta la base de datos PostgreSQL
- Servicio `liquibase` — ejecuta las migraciones al arrancar

Con un solo comando `docker-compose up` el entorno queda listo para trabajar.

## Justificación técnica
- Docker garantiza que el entorno es igual en cualquier máquina.
- El servicio de Liquibase depende del de PostgreSQL, por lo que las migraciones 
  se aplican automáticamente en el orden correcto.
- El archivo `docker-compose.yml` se versiona junto con el proyecto.
- Facilita la incorporación de nuevos integrantes al equipo.

## Consecuencias
- El proyecto incluye un `docker-compose.yml` documentado en la carpeta `docker/`.
- Los desarrolladores usarán contenedores en lugar de instalaciones locales.
- Se podrán agregar nuevos servicios al compose a medida que el proyecto crezca.
- El entorno de desarrollo será estable, reproducible y fácil de mantener.
