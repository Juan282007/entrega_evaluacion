# ADR 005: Implementar Docker Compose para el entorno de desarrollo y despliegue técnico

## Fecha

2026-04-14

## Estado

Aceptado

## Contexto

Para estabilizar el desarrollo y garantizar que las migraciones de base de datos se apliquen en entornos similares, el equipo necesita una forma reproducible de ejecutar los servicios esenciales del proyecto.

## Decisión

Se utilizará Docker Compose como la pieza central del entorno local y de pruebas, orquestando al menos PostgreSQL y Liquibase. Esto permitirá levantar el stack completo con un solo comando.

## Razones

- Docker Compose simplifica la configuración y reduce la variabilidad entre máquinas de desarrollo.
- Permite recrear un entorno de base de datos consistente con volúmenes persistentes.
- Automáticamente gestiona dependencias entre servicios como PostgreSQL y Liquibase.
- Facilita la incorporación de nuevos desarrolladores y el testing local.
- Contribuye a un despliegue técnico más confiable y predecible.

## Consecuencias

- El proyecto debe incluir un `docker-compose.yml` documentado y versionado.
- Las migraciones de Liquibase se ejecutarán dentro del flujo de arranque del contenedor.
- Los desarrolladores trabajarán con contenedores en lugar de instalaciones locales dispersas.
- La infraestructura de desarrollo será más estable y reproducible.
- Se podrán agregar nuevos servicios del sistema en el mismo archivo Compose a medida que el proyecto crezca.
