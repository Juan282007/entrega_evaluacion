# ADR 003: Implementar Liquibase para administrar el versionamiento del DDL

## Fecha

2026-04-14

## Estado

Aceptado

## Contexto

El proyecto requiere una forma confiable de versionar, aplicar y rastrear cambios en el esquema de la base de datos. El control manual de scripts DDL no es suficiente para entornos colaborativos y multi-etapa.

## Decisión

Se implementará Liquibase para administrar el versionamiento del DDL. Todas las modificaciones de esquema deben realizarse mediante changesets en archivos XML, registrados en el changelog maestro.

## Razones

- Liquibase permite versionar los cambios de base de datos junto con el código del proyecto.
- Ofrece capacidad de rollback y control de cambios auditables.
- Facilita la ejecución de migraciones en distintos entornos de manera consistente.
- Es compatible con PostgreSQL y con la orquestación de contenedores en Docker.
- Reduce errores de despliegue causados por scripts DDL incompletos o desincronizados.

## Consecuencias

- Los cambios de esquema solo se aplicarán mediante Liquibase.
- Se mantendrá un changelog maestro que incluya todos los cambios del proyecto.
- El equipo debe ser disciplinado en crear changesets claros y únicos.
- El proceso de despliegue deberá incluir la ejecución de Liquibase antes de levantar la aplicación.
- Se espera mayor estabilidad en la evolución del esquema de datos.
