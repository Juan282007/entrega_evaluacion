# ADR-003: Implementar Liquibase para versionar los cambios del esquema de base de datos

## Fecha
2026-04-14

## Estado
Aceptado

## Contexto
El proyecto parte de un modelo PostgreSQL ya construido en un único script SQL. 
Para poder evolucionar ese modelo de forma ordenada en distintos entornos, se 
necesita una herramienta que controle y registre cada cambio que se haga al esquema.

## Problema
Sin una herramienta de versionamiento, los cambios al esquema se aplican manualmente 
y es imposible saber qué cambios se aplicaron en cada entorno, en qué orden y quién 
los hizo. Esto genera errores difíciles de detectar y ambientes desincronizados.

## Decisión
Se usará Liquibase para gestionar todos los cambios del esquema. Los changelogs 
se organizarán por dominio funcional siguiendo la estructura del proyecto:

- `01_ddl/` para creación de tablas, índices y extensiones
- `02_dml/` para datos iniciales
- `03_dcl/` para roles y permisos
- `05_rollbacks/` para revertir cambios

## Justificación técnica
- Liquibase registra cada cambio aplicado en la tabla `DATABASECHANGELOG`.
- Permite revertir cambios con rollback controlado.
- Es compatible con PostgreSQL y funciona dentro de contenedores Docker.
- Organizar los changelogs por dominio mantiene el proyecto ordenado y fácil de mantener.

## Consecuencias
- Todos los cambios al esquema se harán únicamente a través de Liquibase.
- Se mantendrá un changelog maestro que apunta a los changelogs por dominio.
- El equipo debe crear un changeset por cada cambio, nunca modificar uno ya aplicado.
- El proceso de despliegue siempre incluirá la ejecución de Liquibase antes de arrancar.
