# ADR 001: Ampliar un nuevo dominio funcional relacionado con el sistema existente

## Fecha

2026-04-14

## Estado

Aceptado

## Contexto

El sistema actual de aerolínea ya contiene estructura para datos de vuelos, aeropuertos, tripulaciones y pasajeros. Sin embargo, para aumentar la capacidad del producto y soportar nuevos requerimientos de negocio, es necesario añadir un dominio funcional adicional que complemente el ecosistema actual.

## Decisión

Se ampliará el sistema con un dominio funcional de "Fidelización de Clientes" (Customer Loyalty) que permita gestionar programas de puntos, niveles de cliente, recompensas y el historial de interacciones de pasajeros frecuentes.

## Razones

- Un programa de fidelización es naturalmente compatible con un sistema de aerolínea y aporta valor comercial.
- Aprovecha las entidades existentes de personas, reservas y ventas, reduciendo el costo de integración.
- Mejora la diferenciación del producto y permite futuras funciones de personalización y marketing segmentado.
- Genera datos adicionales para análisis y trazabilidad de comportamiento de pasajeros.

## Consecuencias

- Se agregará una nueva área de datos al modelo relacional, con tablas como `loyalty_program`, `loyalty_account`, `loyalty_transaction` y `loyalty_tier`.
- Se deberán definir migraciones de Liquibase para el nuevo dominio y añadirlas al changelog maestro.
- El equipo deberá mantener documentación del nuevo dominio y su interacción con el sistema existente.
- El proyecto ganará complejidad por el nuevo dominio, pero mejorará la estabilidad comercial y la capacidad de expansión.
