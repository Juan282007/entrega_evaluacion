# ADR 002: Diseñar manejo de roles con permisos diferenciados para fortalecer seguridad y trazabilidad

## Fecha

2026-04-14

## Estado

Aceptado

## Contexto

El sistema necesita garantizar que diferentes usuarios y perfiles tengan acceso restringido según sus responsabilidades, y que las acciones sobre datos sensibles queden registradas para auditoría y control.

## Decisión

Implementar un esquema de control de acceso basado en roles (RBAC) con permisos diferenciados y trazabilidad de acciones. Se definirá una tabla de roles, una tabla de permisos y una tabla de asignación de roles a usuarios.

## Razones

- El RBAC permite separar responsabilidades y minimizar el riesgo de accesos no autorizados.
- Facilita el cumplimiento de políticas de seguridad y de auditoría del sistema.
- Permite mantener una trazabilidad clara de quién realizó qué acción y cuándo.
- Es escalable: se pueden añadir nuevos roles o permisos sin cambiar la lógica de acceso subyacente.
- Mejora la mantenibilidad del sistema al centralizar políticas de seguridad.

## Consecuencias

- Se crean tablas adicionales en la base de datos como `role`, `permission`, `role_permission` y `user_role`.
- Las operaciones críticas deberán validar permisos antes de ejecutarse.
- Se añadirá auditoría de transacciones y registros de acceso para los eventos relevantes.
- El equipo deberá documentar la matriz de permisos y la política de roles.
- El despliegue y la evolución del sistema estarán mejor protegidos frente a cambios no controlados.
